Accounting Assistant Skill

An accounting assistant for the Google Edge AI Gallery that analyzes images of receipts and invoices to extract line items, vendors, and total amounts, securely saving them to a local in-browser database.

Overview

The Accounting Assistant acts as a local processor for your financial documents. By supplying a photo of a receipt or invoice, or directly describing an expense, the AI uses structured data extraction to capture core details like the vendor, total amount, date, and line items. The assistant then stores this information securely in your device's local database using an SQLite abstraction over IndexedDB via WebAssembly (sql.js).

Features

Automated Data Extraction

Transforms pictures of physical receipts/invoices into structured JSON.

Local Embedded Database

Uses a WebAssembly-compiled SQLite database (sql-wasm.js) backed by IndexedDB to ensure your data never improperly leaks to the cloud. All persistent data lives on the edge device.

Web Dashboard Integration

Renders a rich HTML view (assets/webview.html) to display your saved expenses and receipts directly on your mobile device.

How It Works

  1. Input: Send an image of a receipt, an image of an invoice, or text describing an expense to the AI.
  2. Analysis: The AI model processes the input and calls an underlying execution tool (run_js) with the desired extracted parameters:
    • vendor
    • total
    • date
    • items (List of item names, prices, and quantities)
  3. Execution: A script (scripts/index.html) intercepts this structured JSON. It initializes or accesses an existing SQLite table mapped to IndexedDB and executes a SQL INSERT statement with your parsed receipt data.
  4. Display: On successful entry, the tool returns the path to webview.html, which provides an interface to visualize or manage all previously saved expenses.

Requirements

  • This skill requires the Google Edge AI application host.
  • Browser support for IndexedDB and WebAssembly to run the local SQLite database.

Architecture & Layout

  • SKILL.md - Defines the model persona, execution parameters, and strict strictures for the JSON payload required.
  • scripts/index.html - Contains the execution environment where the AI tool passes JSON. Loads sql-wasm.js and provides basic loadFromIndexedDB and saveToIndexedDB abstractions.
  • scripts/sql-wasm.js & scripts/sql-wasm.wasm - The WebAssembly SQLite port.
  • assets/webview.html - The interactive front-end visualizer.
